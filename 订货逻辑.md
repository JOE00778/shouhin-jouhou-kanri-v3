# 订货逻辑文档

> 最终版 · 2026-05-14

---

## 一、适用范围

| 分类 | 条件 | 处理方式 |
|---|---|---|
| **自动订货** | ランク ≠ NEW　且　ランク ≠ 取扱中止　且　実績(30日) > 0 | 每月执行一次，按公式计算 |
| **NEW类** | ランク = NEW | 被动订货，收到需求后才下单 |
| **取扱中止** | ランク = 取扱中止 | 跳过，不订货 |

---

## 二、自动订货计算公式

### 核心计算步骤

```
① 实质库存       = JD在庫 + 弁天在庫 + 発注済
② 下次订货时剩余  = 实质库存 − 実績(30日)
③ 目标库存       = 実績(30日) × 1.5
④ 需要订货量     = 目标库存 − 下次订货时剩余
⑤ 订货箱数       = CEIL( 需要订货量 ÷ ケース入数 )
```

### 计算示例

> 商品：小林製薬 ブレスケア ストロングミント  
> 実績(30日) = 231　／　実質在庫 = 146　／　ケース入数 = 48

```
① 实质库存       = 146
② 下次订货时剩余  = 146 − 231 = −85
③ 目标库存       = 231 × 1.5 = 347
④ 需要订货量     = 347 − (−85) = 432
⑤ 订货箱数       = CEIL(432 ÷ 48) = 9箱
```

## 三、跳过 / 人工确认条件

| 条件 | 处理 |
|---|---|
| ランク = 取扱中止 | ⛔ 跳过，不订货 |
| 実績(30日) = 0 | ⛔ 跳过，无销量不订货 |
| 第④步结果 ≤ 0 | ⛔ 跳过，库存充足 |
| 発注ロット = 0 | ⚠️ 标记异常，人工确认 |
| ケース入数 ÷ 実績(30日) ≥ 2.5 | ⚠️ 箱规过大，人工确认（根据商品具体判断） |

---

## 四、NEW类订货

- **触发条件**：收到客户或内部需求
- **数量计算**：按需求量向上取整到整箱
  ```
  订货箱数 = CEIL( 需求量 ÷ ケース入数 )
  ```

---

## 五、逻辑流程图

```
开始
  │
  ├─ ランク = 取扱中止？ → YES → ⛔ 跳过
  │
  ├─ ランク = NEW？ → YES → 等待需求 → 按需求量订货
  │
  ├─ 実績(30日) = 0？ → YES → ⛔ 跳过
  │
  ├─ ケース入数 ÷ 実績 ≥ 2.5？ → YES → ⚠️ 人工确认
  │
  └─ 执行公式计算
       │
       ├─ 需要订货量 ≤ 0？ → YES → ⛔ 跳过（库存充足）
       │
       ├─ 発注ロット = 0？ → YES → ⚠️ 人工确认
       │
       └─ 输出订货箱数 ✅
```

---

## 六、字段说明

| 字段 | 说明 |
|---|---|
| JD在庫 | JD仓库现有库存 |
| 弁天在庫 | 弁天仓库现有库存 |
| 発注済 | 已订货未到货数量 |
| 実績(30日) | 过去30天销售数量 |
| ケース入数 | 每箱数量 |
| 発注ロット | 最小发注单位 |
