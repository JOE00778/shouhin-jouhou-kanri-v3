## 二、把 Boss 公式翻成工业术语

Boss 2026-05-14 终版公式:

```
推奨月販     = max(平均月販, 直近月販) × トレンド係数(1.2/1.0/0.7)
実質在庫     = JD-物流-千葉 手持 + 注文済(全倉横断)         ← 弁天除外、確保済不扣
必要数       = 推奨月販 × 2.5 − 実質在庫
発注箱数     = CEIL(必要数 / ケース入数)
```

这是工业教科书里的 **Periodic-Review Order-Up-To 策略 = (R, S) policy**:

| Boss 术语 | 工业术语 | 当前取值 |
|---|---|---|
| 「每月执行一次」 | Review interval `R` | **1 月** |
| 推奨月販 | Demand forecast `μ` | `max(avg_4m, latest) × trend` |
| 目標在庫 = 推奨月販 × 1.5 | Order-up-to level `S` | **1.5 月 coverage**(= R=1 + LT≈0.5 + safety≈0,简化形) |
| 必要数 = `S − (実質在庫 − 推奨月販)` | `S − (IP − μ·R)` = `S + μ·R − IP` | 等价于 `2.5×μ − IP` |
| ケース取整 | Lot/case multiple rounding | `case_qty`,无则 `lot_size` 兜底 |

**关键性质**:
- `S − (IP − μ·R) = 2.5μ − IP`(= 教科书 `S + μR − IP` 当 `S = 1.5μ`)
- **没有显式安全库存**(safety_months = 0,所有 buffer 折进系数 1.5 和 trend 1.2)
- **没有显式 lead time**(把 LT 等价折进 1.5 系数;若供应商 LT 大于 ~2 周,缺货风险偏高)
- **取整向上 = 自动产生一个隐式 buffer**(平均 `pack/2` 个量级)

## 三、为什么不选其他经典策略

| 候选 | 理论说明 | 不选的原因 | 备注 |
|---|---|---|---|
| **EOQ**(经济订货批量) | `Q* = √(2DS/H)` 最小化订购成本+持有成本 | 我们 **订购成本 ≈ 0**(月固定流程、无单次启动费),持有成本未量化 → 公式退化 | 仕入先 `min_order_amount` 已替代"启动费"角色 |
| **(s, S) 连续审核** | 在庫降到 `s` 触发,补到 `S` | 我们 review 周期固定 = 1 月,不需要每天监控库存触发 | Page 08 的"在庫+発注済 < 月販×1.2"是 (s, Q) 的退化形,准确率低 → 弃用 |
| **(R, Q) 周期定量** | 每 R 期固定订 Q | Q 固定不应对趋势 → 不符合 Boss 的"按月販×trend 动态调" | — |
| **Newsvendor**(报童) | 单期最优库存,平衡缺货/滞销成本 | 我们 SKU **可滚动**,不是单期消耗品 | 短保质期类(page 19)可考虑,但本次不引入 |
| **Croston / SBA** | 间歇/低频需求专用 | 我们已用 `実績(30日) = 0 → 跳过`,粒度足够 | NEW 类已被动订货,无需此模型 |
| **MRP / 多级 BOM** | 多 echelon 物料推算 | 我们是单级:仕入先 → JD 仓 → 客户 | 不适用 |
| **机器学习预测**(Prophet / LSTM) | 拟合时序 + 加置信区间 | 历史数据 ≤ 12 月,且 2026-02 异常被忽略 → 训练样本不足 | 数据成熟后(≥ 24 月)再评估 |

**结论:Boss 公式 = 简化的 (R, S),在订购成本 ≈ 0 + 销售平稳 + 无显式服务水平要求的场景下是合理选择。** 不需要换算法,需要的是把 4 套实现归一到这一套。

## 四、SKU 分支:4 个状态 → 算法决定

| 状态 | 判定 | 算法分支 | 输出 |
|---|---|---|---|
| **自动订货** | `rank ∉ {NEW, 取扱中止}` 且 `実績(30日) > 0` | Boss (R, S) 公式 | `status='recommended'` / `'needs_review'` |
| **NEW 类** | `rank = NEW` | **被动订货**(等需求来) | `status='new_passive'`,数量=0 出在表里 |
| **取扱中止** | `rank ∈ {取扱中止, メーカー取扱中止}` 或 `handling_status` 同 | 排除 | `n_discontinued_excluded` 计数 |
| **零销量** | `avg_monthly=0` 且 `latest_monthly=0` | 排除 | 静默跳过(不出在表里) |

**人工確認**(出表但需要人看一眼):
- `needs_review_pack`:`case_qty` 和 `lot_size` 都没设(单位 = 1 强算,数字可能离谱)
- `needs_review_oversize`:`pack / rec_monthly ≥ 2.5`(一箱够 2.5 个月以上)

**对外阻塞**:
- `deferred_overstock`:发注后库存月数 > `max_stock_months`(UI 开关,默认关)
- `deferred_min_order`:仕入先合计 < `min_order_amount`

## 八、附:Boss 公式 ↔ 工业 (R, S) 等价证明

设 `R = 1 月`,`μ = 推奨月販`,`IP = 実質在庫 = on_hand + on_order`,`σ = 0`(无显式安全库存)。

教科书 (R, S) 订货量:
```
Q = S − IP_after_review
  = S − (IP − μ·R)
  = S + μ·R − IP
```

Boss 公式直接展开:
```
必要数 = 目標在庫 − 上次発注時剩余
       = (μ·1.5) − (IP − μ)
       = 1.5μ − IP + μ
       = 2.5μ − IP
```

带入教科书形:
```
S + μ·R − IP = S + μ·1 − IP
```

令 `S = 1.5μ`,则:
```
1.5μ + μ − IP = 2.5μ − IP ✓
```

**等价确认**。Boss 公式 = 一个 `S = 1.5×μ` 的 (R=1月, S) 策略,加上 `μ` 用 `max(avg, last) × trend` 估,加上"必要数 ≤ 0 就跳过"的截断。
